ゼロトラスト・アーキテクチャ의 논리적 구성 요소

  1. 1

    常時診断および対応（CDM, Continuous Diagnostics and Mitigation）

  2. 2

    脅威インテリジェンス・フィード（Threat intelligence feed(s)）

  3. 3

    ネットワークおよびシステム活動ログ（Network and system activity logs）

  4. 4

    統合セキュリティイベント管理（SIEM, Security Information and Event Management）

  • 常時診断および対応（CDM）
    PLURA-EDR & WAF
    オペレーティングシステム（OS）のあらゆる情報を収集
    監査ポリシーによるイベントログ
    Web のヘッダーと本文（body）ログの収集により
    適切にパッチ適用された OS を運用しているか
    ハッキングが内部資産まで侵入したかどうか リアルタイム診断と対応
  • 脅威インテリジェンス・フィード（TI）
    PLURA-SIEM
    KISA の CTAS 情報
    PLURA SECaaS XDR Platform で収集した
    攻撃者の情報 総合的に提供


  • ネットワークおよびシステム活動ログ
    PLURA-WAF & Datos & EDR
    Web パケット（本文含む）
    ホストのリソース使用量（CPU、メモリ、ネットワークトラフィック等）
    監査ポリシーにより情報システムのセキュリティ状態をリアルタイム監視
    攻撃者の行為を リアルタイムで活動をロギング
  • 統合セキュリティイベント管理（SIEM）
    PLURA-SIEM
    ネットワーク・情報セキュリティ機器のログ収集
    サーバ・PC のログ収集
    Web・アプリケーションのログ収集
    統合相関分析により異常兆候を検知し、
    MITRE ATT&CK に基づく APT 攻撃の検知
    ハッキング攻撃に対する 先制的な対応を提供
